# 模块三：智能情景对话 -- V1.0 测试用例

> 对应需求：PRD 3.4.2 子模块 3.2 | 版本 V1.0（3 场景 x 5 轮对话）
> 对应设计：HLD 2.2.2.3.2 智能情景对话 + 图2-9 智能情景对话活动图 + 图2-31 智能情景对话时序图
>
> 已实现后端 3 个 API：
> - POST /api/v1/chat/session/start -- 创建会话 + AI 第一轮消息
> - POST /api/v1/chat/message -- 上传音频 + ASR 转写 + 输入过滤 + LLM 回复
> - POST /api/v1/chat/end/{sessionId} -- 结束对话 + 独立评分调用
>
> 已实现前端页面：ConversationSelectPage（场景选择页）、ConversationPage（对话页 + 打字机效果 + Push-to-Talk）、ScoreModal（评分弹窗）
>
> V1.0 特有规则：
> - 3 个场景（self_intro / campus_life / restaurant），每个有独立 Prompt 和角色设定
> - 固定 5 轮对话，第 5 轮用户松手后 AI 回复打字完成 → 自动触发评分
> - 评分权重：语法 40% + 相关性 30% + 流利度 30%

### 9.1 关键契约差异（须确认/修复）

| 编号 | 发现 | 严重程度 | 建议 |
|------|------|----------|------|
| FIND-001 | `POST /api/v1/chat/message` 后端返回 `MessageVO`（round/role/content/createdAt）与前端期望 `AIMessageResponse`（round/userText/aiText/sessionId/totalRounds）字段不对齐。前端 `sendAudioMessage` 中 `response.userText` 将为 undefined，导致用户气泡内容为空。 | **P0 阻塞** | 方案 A：后端 `processMessage()` 返回新增 `MessageWithSessionVO`（含 userText/aiText/sessionId/round/totalRounds 五字段），userText=ASR原文本，aiText=LLM回复内容。方案 B：前端适配 `MessageVO`，在 store 层自行记录 userText（需确认 ASR 原文如何从后端获取） |
| FIND-002 | 后端 `createdAt` 使用 `LocalDateTime` 类型，Jackson 默认序列化为数组格式 `[2026,6,26,10,30,0]` 而非 ISO 8601 字符串。前端 `new Date(msg.createdAt)` 将解析失败。 | **P1 高** | 确认 `application.yml` 中是否配置了 `spring.jackson.serialization.write-dates-as-timestamps=false` + `spring.jackson.date-format=yyyy-MM-dd'T'HH:mm:ss`，或添加 `@JsonFormat` 注解。建议统一使用 `Instant` 类型替代 `LocalDateTime` |
| FIND-003 | 前端 `types/api.ts` 中 `Result<T>` 接口缺少 `timestamp` 字段，但后端 `Result.java` 始终返回 `timestamp`。当前前端 `request()` 仅解构 `result.data` 故无影响，但若后续需要展示时间则需补充。 | P2 低 | 在 `Result<T>` 接口中增加 `timestamp: string` 字段 |
