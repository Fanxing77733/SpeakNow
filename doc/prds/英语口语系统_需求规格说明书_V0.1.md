## 2 总体概述

### 2.1 软件概述

#### 2.1.1 项目介绍

在全球化浪潮下，英语作为国际交流的核心语言，其重要性愈发显著。口语能力作为英语技能的关键一环，直接影响着人们在国际舞台上的交流与合作。然而，传统英语口语学习方式存在诸多弊端：

- 课堂学习时间有限，学生练习机会不足，难以获得一对一的及时指导；
- 市面上部分语言学习软件功能单一，无法满足多样化学习需求；
- 线下培训机构成本高昂，时间和地点受限，难以满足学习者随时随地学习的迫切需求。

基于此，本项目构建一款基于自然语言处理和大语言模型技术的英语口语训练系统，覆盖儿童、青少年、大学生、职场人士和中老年等全年龄段用户，通过 AI 驱动的个性化教学、实时语音评估、情景对话模拟等核心功能，为用户提供全天候、个性化的英语口语训练服务。

本项目为一个全新独立项目，无前序版本。

- **行业领域**：在线教育 / 语言学习 / AI 教育
- **应用方向**：英语口语评测、发音纠错、情景对话练习、自适应学习、语言能力评估
- **核心付费目标用户群**：K12 学生（12-18 岁，中考/高考口语）+ 大学生（四六级口语考试）
- **价值主张**：将"音素级发音评测"与"大模型情景对话"深度融合，形成"练习→评测→纠正→应用"的完整闭环，支持角色扮演和游戏闯关模式提升学习兴趣和粘性

#### 2.1.2 产品环境介绍

本系统为独立 Web 应用，采用前后端分离架构，部署于云服务器。系统整体环境如下所示：

```

│   移动浏览器 (iOS Safari / Android Chrome / 微信内置浏览器)        │

│    用户服务  │  测评服务  │  评测服务  │  对话服务  │  进度服务    │
└──────────────────────────┬───────────────────────────────────────┘
                           │
              ┌────────────┼────────────┬────────────┐
              ▼            ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  MySQL   │ │ ASR API  │ │ 发音评测  │ │  LLM API │
```

- **前端**：React + TypeScript + Tailwind CSS，响应式布局，部署于 CDN
- **后端**：Python FastAPI，通过 AI Gateway 层统一代理所有第三方 API 调用
- **数据库**：MySQL 8.0，存储用户数据、评测记录、对话记录、审计日志
- **第三方 AI 服务**：ASR 语音识别、发音评测、LLM 对话引擎，全部通过后端代理调用，前端不持有 API Key
- **音频资源**：Edge TTS 离线预生成 mp3 文件，部署于静态文件服务

### 3.2 功能模块一：用户端

#### 3.2.1 子功能 用户注册

**1 介绍**

用户通过邮箱或手机号创建账号，并填写年龄和学习目标，用于后续学习内容匹配。此功能不支持第三方社交账号登录（V2 扩展）。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 邮箱 | 用户手动输入 | 1 个 | 注册时必填 | 符合邮箱格式 xxx@xxx.xxx，不超过 100 字符 |
| 手机号 | 用户手动输入 | 1 个 | 注册时必填（与邮箱二选一即可） | 中国大陆手机号 11 位，以 1 开头 |
| 密码 | 用户手动输入 | 1 个 | 注册时必填 | 8-20 位，至少含 1 字母 + 1 数字 |
| 年龄 | 用户选择 | 1 个 | 注册时必填 | 6-99 岁之间 |
| 学习目标 | 用户选择 | 1 个 | 注册时必填 | 三选一：日常交流 / 考试 / 商务 |

操作：用户在注册页面填写以上信息后，点击"注册"按钮提交。

**3 处理**

```
用户点击"注册"按钮
    ↓
前端校验：
  ├── 邮箱格式是否合法？
  │   └── 否 → 提示"请输入正确的邮箱格式"
  ├── 手机号是否为 11 位数字？
  │   └── 否 → 提示"请输入正确的手机号"
  ├── 密码是否 8-20 位 + 含字母和数字？
  │   └── 否 → 提示"密码需8-20位，包含字母和数字"
  ├── 年龄是否在 6-99 之间？
  │   └── 否 → 提示"请选择正确的年龄"
  └── 学习目标是否已选择？
      └── 否 → 提示"请选择学习目标"
    ↓ 前端校验通过
后端校验：
  ├── 邮箱/手机号是否已被注册？
  │   └── 是 → 返回提示"该邮箱/手机号已注册，请直接登录"（含跳转登录页链接）
  └── 数据写入 users 表（密码 bcrypt 哈希存储）
    ↓
注册成功 → 自动跳转到登录页面
```

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 注册成功提示 | 前端页面弹窗 | "注册成功！即将跳转到登录页" |
| 注册失败提示 | 前端页面（表单字段下方） | 具体错误原因（如密码格式错误、邮箱已被占用） |
| 用户数据 | MySQL users 表 | 写入新用户记录 |

#### 3.2.2 子功能 用户登录

**1 介绍**

用户使用已注册的邮箱或手机号 + 密码登录系统，获取 JWT Token 用于后续 API 鉴权。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 账号 | 用户手动输入 | 1 个 | 登录时必填 | 邮箱或手机号 |
| 密码 | 用户手动输入 | 1 个 | 登录时必填 | 8-20 位 |

**3 处理**

```
用户点击"登录"按钮
    ↓
前端校验：账号和密码均不为空
    ↓
后端校验：
  ├── 账号是否存在？
  │   └── 否 → 返回"账号不存在，请先注册"
  ├── 密码是否正确？（bcrypt 比对）
  │   └── 否 → 返回"密码错误，请重试"（连续错误 5 次锁定 30 分钟）
  └── 生成 JWT Token（有效期 7 天）
    ↓
登录成功 → 跳转到首页
```

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 登录成功 | 前端页面跳转 | 跳转至首页 |
| JWT Token | 浏览器 localStorage / cookie | 用于后续 API 请求鉴权 |
| 登录失败提示 | 前端页面 | 错误原因提示 |
| Token 过期处理 | — | 返回 401，前端自动跳转登录页并提示"登录已过期，请重新登录"，保留当前路径以便回跳 |

#### 3.2.3 子功能 英语水平测评

**1 介绍**

用户完成 20 道固定选择题后获得初/中/高三档水平等级，用于匹配合适难度的学习内容。测评涵盖词汇、语法、阅读、听力四个维度。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 答题选择 | 用户点击选项 | 20 题，每题 4 选 1 | 每题限时 60 秒 | A/B/C/D 选项之一 |

操作：用户点击选项提交单题答案，测评页面顶部显示进度条（如"第 5/20 题"）。

**3 处理**

```
用户进入测评页
    ↓
系统按固定顺序出题（词汇 5 题 → 语法 5 题 → 阅读 5 题 → 听力 5 题）
    ↓
每题：
  ├── 60 秒内点击选项 → 记录答案，进入下一题
  └── 超时 → 自动跳过，计 0 分，进入下一题
    ↓
20 题完成后：
  ├── 计算总分（每题 5 分，满分 100 分）
  ├── 0-40 分 → 初级
  ├── 41-70 分 → 中级
  └── 71-100 分 → 高级
    ↓
更新用户画像中的水平等级字段
    ↓
展示测评结果页
```

边界处理：
- 测评中途刷新页面或关闭浏览器：进度不保存，下次进入从头开始
- 不支持返回修改已答题目
- 听力题音频使用 Edge TTS 预生成的 mp3 文件，用户可重复播放

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 测评结果 | 测评结果页 | 展示总分 + 等级（初级/中级/高级）+ 四维得分概要 |
| 水平等级 | MySQL users 表 | 更新用户 level 字段 |

#### 3.2.4 子功能 AI 发音评测与纠错

**1 介绍**

用户看到跟读内容后朗读录音，系统通过 ASR 识别 + 音素级评测引擎对发音进行三维评分（准确度、流利度、完整度），并以逐词颜色标记和音素级纠错提示帮助用户改进发音。此功能为本产品核心差异化功能。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 录音音频 | 用户麦克风录音 | 1 段 | 单次录音 ≤60 秒 | 16kHz Mono WAV，≤5MB |
| 跟读内容 ID | 系统 | 1 个 | — | 预置内容库中的有效 ID |

操作：用户看到跟读文字 → 点击"开始录音"按钮 → 朗读 → 点击"结束录音" → 系统自动上传并评测。

**3 处理**

```
用户点击"结束录音"
    ↓
前端校验：
  ├── 录音时长 <0.5s → 提示"未检测到有效语音，请重新朗读"
  ├── 音量过低 → 提示同上
  └── 音频大小 >5MB → 提示"录音请控制在60秒以内"
    ↓ 前端校验通过
上传音频到后端 → 后端通过 AI Gateway 代理调用 ASR API
    ↓
ASR 返回识别文本 + 时间戳
    ↓
后端调用发音评测 API（传入音频 + 标准文本）
    ↓
评测结果：
  ├── 总分（百分制）
  ├── 准确度评分：音素级发音准确程度
  ├── 流利度评分：语速、停顿频率
  ├── 完整度评分：跟读覆盖度（有无漏词）
  └── 逐词评分 + 音素级纠错信息
    ↓
前端渲染结果：
  ├── 总分大数字 + 三维度分数
  ├── 跟读文本逐词颜色标记
  │   ├── ≥80 分 → 绿色
  │   ├── 60-79 分 → 黄色
  │   └── <60 分 → 红色
  └── 点击红色单词 → 弹出音素纠错面板
      ├── 音素符号（如 /θ/）
      ├── 中文发音描述（如"舌尖轻触上齿，气流从齿缝通过"）
      └── 播放正确发音按钮
    ↓
支持"重新录音" → 清除当前结果 → 回到录音界面
```

边界处理：

| 场景 | 处理 |
| 录音权限未授权 | 弹出引导提示，指引开启浏览器麦克风权限 |
| 录音中被来电/通知中断 | 自动结束录音，已录部分上传评测，提示"录音被中断" |
| 用户中途点击取消 | 清除本地音频缓存，不发起 API 请求，恢复至录音前状态 |
| 快速连续点击录音 | 前端防抖：两次录音间隔 ≥3s |
| 发音评测 API 超时 >10s | 提示"服务繁忙，请稍后重试" |
| 评测 API 返回"无法评分"（噪音过大/非英文） | 提示"环境嘈杂或发音不清，无法准确评分，建议更换安静环境后重试" |
| ASR 返回空文本（置信度低） | 提示"未能识别语音内容，请确保发音清晰后重试" |
| 后端对同一用户 QPS 限流 | 每秒最多 1 次评测请求 |

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 评测结果 | 评测结果页 | 总分 + 三维分 + 逐词颜色标记 + 音素纠错面板 |
| 评测记录 | MySQL practice_records 表 | JSON 存储完整评测结果 |
| 学习时长 | MySQL study_sessions 表 | 记录本次练习录音时长 |
| 错误提示 | 前端页面 | 超时/空识别/无法评分等场景的友好提示 |

#### 3.2.5 子功能 智能语音对话练习

**1 介绍**

用户选择对话场景后，与 AI 虚拟角色进行 5 轮语音对话。每轮对话包含用户语音输入 → ASR 转写 → LLM 生成角色回复 + 评估 → 页面展示文本回复。对话结束后输出语法准确性、回答相关性、流利度三维综合评分。此功能融合了语法纠错和情景角色扮演。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 场景选择 | 用户选择 | 1 个 | 对话开始前 | 5 个场景之一（自我介绍/校园生活/餐厅点餐/旅行出行/商务会议） |
| 难度档位 | 用户选择 | 1 个 | 对话开始前 | 初级/中级/高级 |
| 每轮语音 | 用户麦克风录音 | 5 段 | 每轮 ≤60 秒 | 16kHz Mono WAV，≤5MB |

操作：用户选择场景和难度 → 进入对话 → AI 发起第 1 轮 → 用户语音回复 → AI 回复 → ... → 第 5 轮 AI 回复后自动结束 → 展示评分。

**3 处理**

```
用户选择场景 + 难度 → 进入对话页
    ↓
系统加载场景 Prompt 模板（含角色设定）
    ↓
AI 发起第 1 轮对话（文本 + 角色头像/名称）
    ↓
循环（共 5 轮）：
  ├── 用户点击录音 → 语音输入 → ASR 转写文本
  ├── 转写文本展示在对话气泡中（用户侧）
  ├── 文本发送至 LLM（Prompt = 角色设定 + 难度控制 + 对话历史 + 隐含语法纠错指令）
  ├── LLM 返回角色回复文本 + 语法评价
  ├── 角色回复展示在对话气泡中（AI 侧）
  └── 第 4 轮 AI 回复后 → 底部轻提示"即将结束本次对话，还剩 1 轮"
    ↓
第 5 轮 AI 回复后 → 自动弹出评分弹窗：
  ├── 语法准确性（权重 40%）：时态/主谓一致/冠词等
  ├── 回答相关性（权重 30%）：是否与场景和对话上下文相关
  └── 流利度（权重 30%）：基于 ASR 的停顿/语速信息
    ↓
总分展示 + 维度得分 + 简要评语
```

边界处理：

| 场景 | 处理 |
| 用户说了与场景完全无关的内容 | LLM 在回复中引导回主题（如"I think we got a bit off track. Let's continue with..."） |
| LLM 输出格式不符合预期（JSON 解析失败） | 正则兜底提取；完全失败时降级为"仅对话回复，不评分" |
| LLM 返回敏感/不安全内容 | 兜底回复 + 自动记录异常日志 |
| 对话中网络中断 | 提示"网络不稳定，请重试"，已记录的对话轮次不丢失 |
| 用户说了只有一个单词的回复 | 评分中某维度可能无法计算，降级处理仅输出可计算维度 |
| 录音中断/权限等问题 | 同 3.2.4 发音评测模块的处理方式 |

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 对话记录 | 对话页对话气泡列表 | 5 轮对话的完整展示 |
| 综合评分 | 评分弹窗 | 总分 + 三维度分 + 简要评语 |
| 对话记录 | MySQL conversation_sessions + conversation_messages 表 | 含 status 字段标记异常（completed/asr_failed/llm_timeout/llm_sensitive） |
| 学习时长 | MySQL study_sessions 表 | 记录本次对话录音总时长 |

#### 3.2.6 子功能 学习进度可视化

**1 介绍**

用户可查看自己的学习统计数据，系统以折线图和数字卡片形式展示练习趋势、分数变化和学习总览。

**2 输入**

无用户输入。数据来源于用户历史练习记录。

**3 处理**

```
用户进入"学习进度"页
    ↓
后端查询用户历史数据：
  ├── practice_records 表（评测记录）
  ├── conversation_sessions 表（对话记录）
  └── study_sessions 表（学习时长记录）
    ↓
聚合计算：
  ├── 练习次数 = 跟读评测完成次数 + 对话场景完成次数（测评不计入）
  ├── 总学习时长 = 跟读录音时长总和 + 对话录音时长总和
  └── 历史最高分 = MAX(评测总分)
    ↓
按日聚合练习次数 → 折线图 1
按时序取每次评测分数 → 折线图 2
    ↓
前端渲染（Recharts/ECharts）
```

统计数据口径：
- 练习次数仅统计"已完成"的跟读和对话，不包含取消/中断
- 学习时长不含页面停留时间，仅计算实际录音时长
- 测评不纳入练习次数和学习时长统计

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 练习次数趋势折线图 | 进度页上半部分 | X 轴 = 日期，Y 轴 = 当日练习次数 |
| 发音分数趋势折线图 | 进度页中部 | X 轴 = 练习序号，Y 轴 = 评分分数 |
| 数字卡片 × 3 | 进度页顶部 | 总练习次数 / 总学习时长（分钟）/ 历史最高分 |

## 4 性能需求

### 4.1 时间性能需求

| 指标 | 目标值 | 测量方法 | 说明 |
| 页面首屏加载 | 移动端 Slow 4G：FCP <1.5s、LCP <2.5s；桌面端 WiFi：FCP <0.8s、LCP <1.5s | Lighthouse 测量 | 首屏不含音频文件，音频按需加载 |
| 评测结果返回 | P95 <5s，P99 <8s | 计时口径：录音上传完成时刻 → 评分结果渲染完成时刻 | 采用渐进式加载（先展示文字转写，再加载评分） |
| 对话单轮回复 | P95 <3s，P99 <5s | 计时口径：ASR 识别完成时刻 → LLM 文本渲染完成时刻 | |
| 发音评测 API 调用成功率 | >95% | 以自然日为单位统计 | |
| 录音上传 | 5MB 文件在 4G 网络下 <8s | — | 弱网（Slow 3G）下 <20s |

### 4.2 系统开放性需求

- **可扩充性**：采用前后端分离架构，后端 API 基于 RESTful 规范设计，前端通过标准 HTTP 协议通信，后续新增功能模块无需改动架构
- **可移植性**：前端基于 Web 标准（HTML5/JS/CSS3），可在任何现代浏览器上运行；后端基于 Python FastAPI，可部署于 Linux/Windows 服务器
- **AI 供应商可替换性**：通过 AI Gateway adapter 层隔离第三方服务差异，切换供应商仅修改 adapter 层代码，不影响业务逻辑

### 4.3 界面友好性需求

**通用要求**：
- 响应式设计，适配 375px-1440px 宽度范围
- 色彩方案使用 ui-ux-pro-max 技能设计，采用教育/学习类应用配色
- 所有可点击元素最小触摸区域 44×44px（移动端）
- 加载状态使用骨架屏或 loading 动画，避免白屏等待
- 错误提示使用通俗语言，避免技术术语

**注册/登录页**：
- 简洁背景，突出表单区域
- 密码字段支持显示/隐藏切换
- 登录失败提示使用友好文案

**首页**：
- 展示用户学习总览（三个数字卡片）
- 底部导航栏（移动端）/ 侧边导航栏（桌面端）
- 清晰的功能入口：开始测评、开始练习、场景对话、我的进度

**发音评测页**：
- 跟读文本居中大字展示
- 录音按钮位于底部、拇指可触及区域（移动端建议长按录音模式）
- 录音中波形动画/呼吸灯效果
- 评测结果页：总分大数字 + 三维分 + 逐词高亮，支持向下滚动查看详情

**对话页**：
- 类聊天界面布局，用户气泡右对齐、AI 气泡左对齐
- AI 角色头像 + 名称
- 底部录音按钮
- 第 4 轮对话后出现"还剩 1 轮"轻提示

**学习进度页**：
- 顶部数字卡片横向排列（移动端纵向堆叠）
- 折线图支持缩放和拖拽

### 4.4 系统可用性需求

- 系统 7×24 小时可用，计划内维护时间提前公告
- 非第三方 API 故障场景下，核心服务可用性 >99.5%
- 第三方 API 故障时，评测和对话功能降级提示，不导致页面崩溃

### 4.5 可管理性需求

- 所有 API 调用的请求/响应日志可追溯（包含请求时间、耗时、错误码）
- 审计日志 append-only，不可篡改
- 语音数据到期自动清理，清理日志写入审计记录
- 后端代码通过 ESLint/Prettier/mypy 自动检查

| 2 | 发音评测 API（驰声/先声/科大讯飞） | 音素级发音评测，返回逐词评分和音素纠错信息 | 最新稳定版 | 待选型确认 |
| 3 | LLM API（DeepSeek/通义千问/GPT-4o-mini） | 对话生成 + 语法纠错 + 综合评分 | 最新稳定版 | DeepSeek / 阿里 / OpenAI |

**接口调用方式**：所有第三方 API 调用均由后端 AI Gateway 层代理，前端不直接调用第三方服务。

### 5.3 硬件接口

本系统为纯软件系统，无特殊硬件接口需求。用户设备需具备：

- 麦克风（用于录音）
- 扬声器/耳机（用于播放标准发音音频）
- 网络连接

### 5.4 通讯接口

- 前端与后端之间通过 HTTPS + RESTful API 通信，数据格式为 JSON
- 后端与第三方 AI API 之间通过 HTTPS 通信
- Web Audio API 要求页面运行在安全上下文（HTTPS 或 localhost）
- 音频文件上传使用 multipart/form-data 编码

## 7 软件质量特性

### 7.1 可靠性

- **适应性**：前后端分离架构支持在原有功能基础上扩充新模块，不影响原系统架构。兼容 Desktop Chrome/Safari/Firefox/Edge 近 2 个大版本、iOS Safari、Android Chrome、微信内置浏览器。
- **容错性**：第三方 API 超时/故障时前端展示友好降级提示，不导致页面崩溃或白屏。数据库连接池 + 自动重连机制。服务崩溃后自动重启（systemd/Docker 守护进程）。
- **可恢复性**：网络恢复正常后系统自动恢复服务能力。对话中断后已有消息不丢失，网络恢复后可查看历史消息。音频上传失败支持自动重试（最多 3 次）。

### 7.2 易用性

- 界面设计清晰简洁，核心功能入口醒目
- 阻止用户输入非法数据：前端实时校验 + 后端二次校验
- 关键操作（如退出对话、重录）提供二次确认
- 复杂流程（测评、对话）提供进度指示
- 移动端优先的交互设计：录音按钮置于拇指区、长按录音避免误操作
- 错误提示使用通俗语言："未能识别语音内容，请确保发音清晰、环境安静后重试"，而非"ASR confidence < 0.3"
- 首次使用可提供简要引导提示（V2 扩展为完整引导流程）
