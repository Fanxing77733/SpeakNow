## 2 总体概述

### 2.1 软件概述

#### 2.1.1 项目介绍

在全球化浪潮下，英语作为国际交流的核心语言，其重要性愈发显著。口语能力作为英语技能的关键一环，直接影响着人们在国际舞台上的交流与合作。然而，传统英语口语学习方式存在诸多弊端：

- 课堂学习时间有限，学生练习机会不足，难以获得一对一的及时指导；
- 市面上部分语言学习软件功能单一，无法满足多样化学习需求；
- 线下培训机构成本高昂，时间和地点受限，难以满足学习者随时随地学习的迫切需求。

基于此，本项目构建一款基于自然语言处理和大语言模型技术的英语口语训练系统，覆盖儿童、青少年、大学生、职场人士和中老年等全年龄段用户，通过 AI 驱动的个性化教学、实时语音评估、情景对话模拟等核心功能，为用户提供全天候、个性化的英语口语训练服务。

本项目为一个全新独立项目，无前序版本。

- **行业领域**：在线教育 / 语言学习 / AI 教育
- **应用方向**：英语口语评测、发音纠错、情景对话练习、自适应学习、语言能力评估
- **核心付费目标用户群**：K12 学生（12-18 岁，中考/高考口语）+ 大学生（四六级口语考试）
- **价值主张**：将"音素级发音评测"与"大模型情景对话"深度融合，形成"练习→评测→纠正→应用"的完整闭环，支持角色扮演和游戏闯关模式提升学习兴趣和粘性

#### 2.1.2 产品环境介绍

本系统为独立 Web 应用，采用前后端分离架构，部署于云服务器。系统整体环境如下所示：

```

│                          用户终端                                │
│   桌面浏览器 (Chrome/Safari/Firefox/Edge)                        │
│   移动浏览器 (iOS Safari / Android Chrome / 微信内置浏览器)        │
└──────────────────────────┬──────────────────────────────────────┘
                           │  HTTPS
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│                    前端 (React SPA)                              │
│   React 18 + TypeScript + Tailwind CSS + Zustand + React Router  │
│   注册登录  │  水平测评  │  发音跟读  │  智能对话  │  学习进度   │
└──────────────────────────┬───────────────────────────────────────┘
                           │  REST API (/api/v1/*)
                           ▼
┌──────────────────────────────────────────────────────────────────┐
│              后端 API 服务 (FastAPI + SQLAlchemy 2.0)            │
│    用户服务  │  测评服务  │  AI Gateway  │  对话服务  │  进度服务  │
└──────────────────────────┬───────────────────────────────────────┘
                           │
              ┌────────────┼────────────┬────────────┐
              ▼            ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  MySQL   │ │ ASR API  │ │ 发音评测  │ │  LLM API │
        │  8.0     │ │(腾讯/阿里)│ │(驰声/讯飞)│ │(DeepSeek)│
        └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

- **前端**：React 18 + TypeScript + Tailwind CSS + Zustand（状态管理）+ React Router v6（路由），响应式布局，部署于 CDN **[SA]**
- **后端**：Python FastAPI + SQLAlchemy 2.0 + Alembic（数据库迁移），通过 AI Gateway 层统一代理所有第三方 API 调用 **[SA]**
- **数据库**：MySQL 8.0，存储用户数据、评测记录、对话记录、审计日志
- **第三方 AI 服务**：ASR 语音识别、发音评测、LLM 对话引擎，全部通过后端 AI Gateway 代理调用，前端不持有 API Key
- **音频资源**：Edge TTS 离线预生成 mp3 文件，部署于静态文件服务
- **图表库**：Recharts（React 原生图表库）用于学习进度可视化 **[SA]**
- **前端音频处理**：MediaRecorder API + recorder-core（备选），微信环境需增加格式检测 + WebM→WAV 转码 **[SA][QA]**
- **API 版本化**：所有 API 路径使用 `/api/v1/` 前缀 **[SA]**

### 3.2 功能模块一：用户端

#### 3.2.1 子功能 用户注册

**1 介绍**

用户通过邮箱或手机号创建账号，并填写年龄和学习目标，用于后续学习内容匹配。此功能不支持第三方社交账号登录（V2 扩展）。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 邮箱 | 用户手动输入 | 1 个 | 注册时必填 | 符合邮箱格式（RFC 5322 标准，含 `+` 号等合法字符），不超过 100 字符 **[QA]** |
| 手机号 | 用户手动输入 | 1 个 | 注册时必填（与邮箱二选一即可） | 中国大陆手机号 11 位，以 1 开头 |
| 密码 | 用户手动输入 | 1 个 | 注册时必填 | 8-20 位，至少含 1 字母 + 1 数字 |
| 年龄 | 用户选择 | 1 个 | 注册时必填 | 6-99 岁之间（DB CHECK 约束 `age BETWEEN 6 AND 99`） |
| 学习目标 | 用户选择 | 1 个 | 注册时必填 | 三选一：日常交流 / 考试 / 商务 |

操作：用户在注册页面填写以上信息后，点击"注册"按钮提交。

**3 处理**

```
用户点击"注册"按钮
    ↓
前端校验：
  ├── 邮箱格式是否合法？（RFC 5322 标准）  **[QA]**
  │   └── 否 → 提示"请输入正确的邮箱格式"
  ├── 手机号是否为 11 位数字？
  │   └── 否 → 提示"请输入正确的手机号"
  ├── 密码是否 8-20 位 + 含字母和数字？
  │   └── 否 → 提示"密码需8-20位，包含字母和数字"
  ├── 年龄是否在 6-99 之间？
  │   └── 否 → 提示"请选择正确的年龄"
  └── 学习目标是否已选择？
      └── 否 → 提示"请选择学习目标"
    ↓ 前端校验通过
后端校验：
  ├── 邮箱/手机号是否已被注册？
  │   └── 是 → 返回 409，提示"该邮箱/手机号已注册，请直接登录"（含跳转登录页链接）
  ├── 同一 IP 每分钟注册请求 ≤3 次？  **[SA][QA]**
  │   └── 否 → 返回 429，提示"操作过于频繁，请稍后再试"
  └── 数据写入 users 表（密码 bcrypt 哈希存储，cost factor=12）
    ↓
注册成功 → 自动跳转到登录页面
```

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 注册成功提示 | 前端页面弹窗 | "注册成功！即将跳转到登录页" |
| 注册失败提示 | 前端页面（表单字段下方） | 具体错误原因（如密码格式错误、邮箱已被占用） |
| 用户数据 | MySQL users 表 | 写入新用户记录 |

#### 3.2.2 子功能 用户登录

**1 介绍**

用户使用已注册的邮箱或手机号 + 密码登录系统，获取 JWT Token 用于后续 API 鉴权。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 账号 | 用户手动输入 | 1 个 | 登录时必填 | 邮箱或手机号 |
| 密码 | 用户手动输入 | 1 个 | 登录时必填 | 8-20 位 |

**3 处理**

```
用户点击"登录"按钮
    ↓
前端校验：账号和密码均不为空
    ↓
后端校验（全局 QPS 限流：同一 IP 每分钟登录请求 ≤10 次）：  **[SA][QA]**
  ├── 账号是否存在？
  │   └── 否 → 返回"账号不存在，请先注册"
  ├── 密码是否正确？（bcrypt 比对）
  │   └── 否 → 返回"密码错误，请重试"（连续错误 5 次锁定 30 分钟，第 6 次起返回"账号已被临时锁定，请 30 分钟后再试"）  **[QA]**
  └── 生成 JWT Token（有效期 7 天）
    ↓
登录成功 → 跳转到首页
```

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 登录成功 | 前端页面跳转 | 跳转至首页 |
| JWT Token | httpOnly + Secure + SameSite=Strict Cookie（生产环境）；可暂用 localStorage（开发阶段，上线前替换） **[SA]** | 用于后续 API 请求鉴权 |
| 登录失败提示 | 前端页面 | 错误原因提示 |
| Token 过期处理 | — | 返回 401，前端自动跳转登录页并提示"登录已过期，请重新登录"，保留当前路径以便回跳。**若用户在录音/对话中 Token 过期，前端需在跳转前将已录音频暂存至 IndexedDB，登录成功后恢复** **[QA]** |

#### 3.2.3 子功能 英语水平测评

**1 介绍**

用户完成 20 道固定选择题后获得初/中/高三档水平等级，用于匹配合适难度的学习内容。测评涵盖词汇、语法、阅读、听力四个维度。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 答题选择 | 用户点击选项 | 20 题，每题 4 选 1 | 每题限时 60 秒 | A/B/C/D 选项之一 |

操作：用户点击选项提交单题答案，测评页面顶部显示进度条（如"第 5/20 题"）。

**3 处理**

```
用户进入测评页
    ↓
后端校验：assessment_questions 表记录数 ≥20 条？  **[QA]**
  └── 否 → 返回错误，前端展示"题库维护中，请稍后再试"
    ↓
系统按固定顺序出题（词汇 5 题 → 语法 5 题 → 阅读 5 题 → 听力 5 题）
    ↓
每题：
  ├── 60 秒内点击选项 → 记录答案，进入下一题
  └── 超时 → 自动跳过，计 0 分，进入下一题
    ↓ 最后 10 秒倒计时变红  **[QA]**
20 题完成后：
  ├── 计算总分（每题 5 分，满分 100 分）
  ├── 0-40 分 → 初级
  ├── 41-70 分 → 中级
  └── 71-100 分 → 高级
    ↓
更新用户画像中的水平等级字段（首次测评自动写入，后续手动测评需用户确认更新）
    ↓
展示测评结果页
```

边界处理：
- 测评中途刷新页面或关闭浏览器：进度不保存，下次进入从头开始
- 不支持返回修改已答题目
- 听力题音频使用 Edge TTS 预生成的 mp3 文件，用户可重复播放
- 后端 API 返回题目时，**必须对字段做白名单过滤，仅返回 question_text + options_json + sort_order，不得返回 correct_answer** **[SA]**

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 测评结果 | 测评结果页 | 展示总分 + 等级（初级/中级/高级）+ 四维得分概要 |
| 水平等级 | MySQL users 表 | 更新用户 level 字段 |

#### 3.2.4 子功能 AI 发音评测与纠错

**1 介绍**

用户看到跟读内容后朗读录音，系统通过 ASR 识别 + 音素级评测引擎对发音进行三维评分（准确度、流利度、完整度），并以逐词颜色标记和音素级纠错提示帮助用户改进发音。此功能为本产品核心差异化功能。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 录音音频 | 用户麦克风录音 | 1 段 | 单次录音 ≤60 秒 | 16kHz Mono WAV/WebM（前端按需转码），≤5MB |
| 跟读内容 ID | 系统 | 1 个 | — | 预置内容库中的有效 ID |

操作：用户看到跟读文字 → 点击"开始录音"按钮 → 朗读 → 点击"结束录音" → 系统自动上传并评测。移动端建议使用**长按录音（Push-to-Talk）**模式。

**3 处理**

```
用户点击"结束录音"（或移动端松手结束长按）
    ↓
前端校验：
  ├── 录音时长 <0.5s → 提示"未检测到有效语音，请重新朗读"
  ├── 音量过低 → 提示同上
  ├── 录音时长 >60s → 前端自动截断，提示"录音已自动结束（最长60秒）"  **[QA]**
  │   └── 校验优先级：先检测时长（前端本地），再检测大小（上传前）  **[QA]**
  └── 音频大小 >5MB → 提示"录音请控制在60秒以内"
    ↓ 前端校验通过
后端 session 级录音状态锁检查：同一用户同一 content_id 是否有进行中的录音？  **[SA][QA]**
  └── 是 → 返回 429，提示"上一段录音正在处理中，请稍后"
    ↓
上传音频到后端 → 后端通过 AI Gateway 代理调用 ASR API
    ↓
ASR 返回识别文本 + 时间戳
    ↓
后端调用发音评测 API（传入音频 + 标准文本）
    ↓
评测结果：
  ├── 总分（百分制）
  ├── 准确度评分：音素级发音准确程度
  ├── 流利度评分：语速、停顿频率
  ├── 完整度评分：跟读覆盖度（有无漏词）
  └── 逐词评分 + 音素级纠错信息
    ↓
前端渲染结果：
  ├── 总分大数字 + 三维度分数
  ├── 跟读文本逐词颜色标记
  │   ├── ≥80 分 → 绿色
  │   ├── 60-79 分 → 黄色
  │   └── <60 分 → 红色
  └── 点击红色单词 → 弹出音素纠错面板
      ├── 音素符号（如 /θ/）
      ├── 中文发音描述（如"舌尖轻触上齿，气流从齿缝通过"）
      └── 播放正确发音按钮
    ↓
支持"重新录音" → 清除当前结果 → 回到录音界面
```

边界处理：

| 场景 | 处理 | 来源 |
| 录音权限未授权 | 弹出引导提示，指引开启浏览器麦克风权限 | — |
| 录音中被来电/通知中断 | 自动结束录音，已录部分上传评测，提示"录音被中断，已将已录部分提交评测" | — |
| 用户中途点击取消 | 清除本地音频缓存，不发起 API 请求，恢复至录音前状态 | — |
| 快速连续点击录音 | 前端防抖：两次录音间隔 ≥3s | — |
| **录音中浏览器标签页切到后台（移动端）** | 前端监听 Page Visibility API，给予 3 秒缓冲后自动截断录音并上传已录部分 | **[QA]** |
| 发音评测 API 超时 >10s | 提示"服务繁忙，请稍后重试"，practice_records.status 写入 timeout | — |
| 评测 API 返回"无法评分"（噪音过大/非英文） | 提示"环境嘈杂或发音不清，无法准确评分，建议更换安静环境后重试"，status 写入 eval_failed | — |

| 后端对同一用户 QPS 限流 | 每秒最多 1 次评测请求，超过返回 429 + "操作过于频繁，请稍后再试" | — |
| **微信内置浏览器录音格式不兼容** | 前端检测 MediaRecorder.isTypeSupported('audio/wav')，不支持则使用 WebM → 服务端 ffmpeg 转码 WAV **[SA]** | **[SA][QA]** |

| 学习时长 | MySQL study_sessions 表 | 记录本次练习录音时长 |
| 错误提示 | 前端页面 | 超时/空识别/无法评分等场景的友好提示 |

#### 3.2.5 子功能 智能语音对话练习

**1 介绍**

用户选择对话场景后，与 AI 虚拟角色进行 5 轮语音对话。每轮对话包含用户语音输入 → ASR 转写 → LLM 生成角色回复 + 评估 → 页面展示文本回复。对话结束后输出语法准确性、回答相关性、流利度三维综合评分。此功能融合了语法纠错和情景角色扮演。

**2 输入**

| 输入项 | 来源 | 数量 | 时间要求 | 有效输入范围 |
| 场景选择 | 用户选择 | 1 个 | 对话开始前 | **3 个场景之一（自我介绍 / 校园生活 / 餐厅点餐），旅行出行和商务会议 → V1.1** **[SA]** |
| 难度档位 | 用户选择 | 1 个 | 对话开始前 | 初级/中级/高级 |
| 每轮语音 | 用户麦克风录音 | 5 段 | 每轮 ≤60 秒 | 16kHz Mono WAV，≤5MB |

操作：用户选择场景和难度 → 进入对话 → AI 发起第 1 轮 → 用户语音回复 → AI 回复 → ... → 第 5 轮 AI 回复后自动结束 → 展示评分。

**3 处理**

```
用户选择场景 + 难度 → 进入对话页
    ↓
后端校验：同一用户当前没有其他 active 对话会话  **[SA]**
  └── 是 → 返回 429，提示"您有一个进行中的对话，请先完成"
    ↓
系统加载场景 Prompt 模板（含角色设定）
    ↓
AI 发起第 1 轮对话（文本 + 角色头像/名称）
    ↓
循环（共 5 轮）：
  ├── 用户点击录音 → 语音输入 → ASR 转写文本
  │   └── ASR 失败或返回空文本 → 展示错误气泡"语音识别失败，请重试本轮"，不计入总轮数  **[QA]**
  ├── 转写文本展示在对话气泡中（用户侧）
  ├── 文本发送至 LLM
  │   └── 输入过滤：长度限制 ≤500 字符 + 敏感模式正则检测（防 Prompt Injection） **[SA]**
  ├── LLM 返回角色回复文本（本轮不嵌入评分指令，评分在对话结束后独立调用） **[SA]**
  ├── 角色回复展示在对话气泡中（AI 侧）
  ├── LLM 返回 429（rate limit/quota exceeded）→ 提示"服务繁忙，请稍后再试"，已完轮次保留  **[QA]**
  ├── LLM 返回 5xx → 同超时处理，status 写入 llm_timeout  **[QA]**
  └── 第 4 轮 AI 回复后 → 底部轻提示"即将结束本次对话，还剩 1 轮"
    ↓
第 5 轮 AI 回复后 → 自动弹出评分弹窗：
  ├── 独立调用评分 Prompt（Temperature=0.1-0.3），传入完整对话历史 **[SA]**
  ├── 语法准确性（权重 40%）：时态/主谓一致/冠词等
  ├── 回答相关性（权重 30%）：是否与场景和对话上下文相关
  └── 流利度（权重 30%）：基于 ASR 的停顿/语速信息
    ↓
总分展示 + 维度得分 + 简要评语
    ↓
中间轮次 LLM 失败时的评分策略：  **[QA]**
  ├── 如 5 轮全部成功 → 正常三维评分
  ├── 如仅 1-2 轮失败 → 已完成轮次评分保留，失败轮次标注"未计算"
  └── 如 ≥3 轮失败 → 对话作废，提示"本次对话有效轮次不足，无法评分"
```

边界处理（发音评测中已有的录音相关边界同样适用，此处补充对话特有场景）：

| 场景 | 处理 | 来源 |
| 用户说了与场景完全无关的内容 | LLM 在回复中引导回主题（如"I think we got a bit off track. Let's continue with..."） | — |
| LLM 输出格式不符合预期（JSON 解析失败） | 正则兜底提取评分字段；完全失败时降级为"仅展示对话文本，不输出评分"并日志标记 llm_parse_failed | — |
| LLM 返回敏感/不安全内容 | 兜底替换为"Mmm, let's change the topic a bit." + 自动记录异常日志（action=sensitive_content_blocked） **[SA]** | **[SA]** |
| 对话中网络中断 | 提示"网络不稳定，请重试"；已完成的轮次消息不丢失（前端 IndexedDB 暂存）；网络恢复后可继续 | — |
| 用户说了只有一个单词的回复 | 评分中某维度可能无法计算，降级处理仅输出可计算维度 | — |
| **LLM API 返回 429（rate limit/quota exceeded）** | 区分于 5xx：提示"服务繁忙，请稍后再试"，已完轮次保留 **[QA]** | **[QA]** |
| **用户主动退出对话（中间轮次）** | 弹窗确认"确定退出吗？已完成的对话将被保存"，确认后 status=user_aborted，已完成轮次写入 DB **[QA]** | **[QA]** |

**4 输出**

| 输出项 | 输出到何处 | 说明 |
| 对话记录 | 对话页对话气泡列表 | 5 轮对话的完整展示 |
| 综合评分 | 评分弹窗 | 总分 + 三维度分 + 简要评语 |
| 对话记录 | MySQL conversation_sessions + conversation_messages 表 | 含 status 字段标记异常 |
| 学习时长 | MySQL study_sessions 表 | 记录本次对话录音总时长 |

## 7 软件质量特性

### 7.1 可靠性

- **适应性**：前后端分离架构支持在原有功能基础上扩充新模块，不影响原系统架构。兼容 Desktop Chrome/Safari/Firefox/Edge 近 2 个大版本、iOS Safari、Android Chrome、微信内置浏览器。
- **容错性**：第三方 API 超时/故障时前端展示友好降级提示，不导致页面崩溃或白屏。数据库连接池 + 自动重连机制。服务崩溃后自动重启（systemd/Docker 守护进程）。**前端录音文件离线暂存至 IndexedDB，网络恢复后自动重试上传（最多 3 次）** **[SA]**。
- **可恢复性**：网络恢复正常后系统自动恢复服务能力。对话中断后已有消息不丢失（IndexedDB 暂存），网络恢复后可查看历史消息。音频上传失败支持自动重试。

### 7.2 易用性

- 界面设计清晰简洁，核心功能入口醒目
- 阻止用户输入非法数据：前端实时校验 + 后端二次校验
- 关键操作（如退出对话、重录）提供二次确认
- 复杂流程（测评、对话）提供进度指示
- 移动端优先的交互设计：录音按钮置于拇指区、长按录音避免误操作
- 错误提示使用通俗语言，**禁止使用技术术语（如"confidence < 0.3"、"HTTP 500"、"timeout"、"JSON parse error"），已批准的友好文案清单如下** **[QA]**：

| 错误场景 | 友好提示文案 |
| 录音 <0.5s / 音量过低 | "未检测到有效语音，请重新朗读" |
| ASR 空文本 / 置信度低 | "未能识别语音内容，请确保发音清晰、环境安静后重试" |
| 发音评测 API 超时 | "服务繁忙，请稍后重试" |
| 发音评测无法评分（噪音/非英文） | "环境嘈杂或发音不清，无法准确评分，建议更换安静环境后重试" |
| 评测 QPS 超限 | "操作过于频繁，请稍后再试" |
| 网络中断 | "网络不稳定，请重试" |
| Token 过期 | "登录已过期，请重新登录" |
| 题库维护中 | "题库维护中，请稍后再试" |
| LLM 服务繁忙 | "服务繁忙，请稍后再试" |

- 首次使用可提供简要引导提示（V2 扩展为完整引导流程）
