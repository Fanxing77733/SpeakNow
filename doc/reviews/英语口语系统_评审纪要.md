## 三、SA 技术评审要点

### 3.1 技术栈评估

**明确的部分**（良）：React + TS + Tailwind / FastAPI / MySQL 8.0 / AI Gateway 层 — 均合理。

**缺失的技术选型**（需在阶段 1 前补全）：

| 缺失项 | SA 建议 | PM 决议 |
| 前端状态管理 | Zustand（轻量） | 采纳 |
| 前端路由 | React Router v6 | 采纳 |
| 后端 ORM | SQLAlchemy 2.0 + Alembic | 采纳 |
| 图表库 | Recharts 或 ECharts | 采纳 |
| 前端音频处理 | recorder-core 或原生 MediaRecorder + ffmpeg.wasm | 阶段 1 验证后定 |
| 静态文件存储 | 本地磁盘（MVP）+ MinIO/OSS（V2） | 采纳 |
| 日志系统 | Python logging → MySQL + ELK（V2） | 采纳 |

### 3.2 工作量评估（核心关注点）

SA 自底向上估算：

| 阶段 | 乐观（人天） | 现实（人天） |
| 阶段 1（初始化 + 探路） | 6.5 | 10.5 |
| 阶段 2（核心功能） | 14 | 21 |
| 阶段 3（联调打磨） | 8.5 | 13.5 |
| **合计** | **29** | **45** |

**结论**：即便最乐观估计也超出 22 天 32%，现实估计超出 100%。**必须裁剪功能。**

#### PM 建议的功能裁剪方案

| 方案 | 内容 | 节省人天 | 影响评估 |
| **A（推荐）** | 对话场景 5→3 个（保留自我介绍/校园生活/餐厅点餐，其余 V1.1） | 2-3 天 | 低影响，3 场景已能验证闭环 |
| **B（推荐）** | 进度页折线图暂缓，V1.0 仅展示 3 张数字卡片 | 1-2 天 | 低影响，数字卡片足以展示核心数据 |
| **C（可选）** | 内容数据削减：单词 250→100、句子 90→40 | 1-2 天 | 低影响，内容量 V1.1 再扩充 |
| **D（不推荐）** | 延期测评模块，默认分配"中级" | 2-3 天 | 高影响，破坏用户体验闭环 |
| **E（不推荐）** | 仅桌面端不做响应式 | 2-3 天 | 高影响，核心用户以手机为主 |

**PM 决议：执行 A + B（节省 3-5 天），缩小与 22 天目标的差距。若仍不足，加 C。**

## 四、QA 质量评审要点

### 4.1 验收标准可测试性

**主要问题**：第 8 章 30 条 REQ-ID 做了 A/B/C 分类，但每条 REQ 无独立显式 AC。功能细节散落在第 3 章各处，测试人员需自行拼凑。

**4 章性能指标**：整体可度量，但 3 处待细化：
- P95/P99 缺最小样本量（建议 N≥100）和测量周期
- "成功率 >95%" 中"成功"的判定口径未定义
- 录音上传耗时缺测量工具和重试次数

### 4.2 遗漏的异常场景（6 项）

| # | 遗漏模块 | 遗漏场景 |
| 1 | 注册 | 邮箱含 `+` 号（RFC 5322 合法）被误判；批量注册无频率限制 |
| 2 | 登录 | 录音/对话中 Token 过期 → 仅跳转登录页，丢失进行中的练习数据 |
| 3 | 测评 | 后端题库 <20 条时前端无降级提示 |
| 4 | 对话 | LLM 返回 429（rate limit/quota exceeded）与 5xx 未区分处理 |
| 5 | 对话 | 中间轮次 LLM 失败时评分策略未定义 |
| 6 | 进度 | 新用户 0 条记录时进度页展示未定义 |

### 4.3 遗漏的边界条件（8 项）

| # | 类别 | 遗漏边界 |
| 1 | 并发 | 同一用户多标签页同时录音无状态锁 |
| 2 | 存储 | 音频文件累计磁盘空间无上限告警 |
| 3 | 弱网 | ASR/LLM 在弱网下的超时和重试策略未定义 |
| 4 | 浏览器 | 微信内置浏览器 MediaRecorder 格式限制 |
| 5 | 音频极值 | 60s 和 5MB 两个边界同时校验的优先级未定义 |
| 6 | 数据库 | CHECK 约束（如 age 6-99、TINYINT 范围）未声明 |
| 7 | 时区 | DATETIME 无时区可能导致按日聚合偏移 |
| 8 | 音频删除 | "7 天"是自然日还是 168 小时精确删除未定义 |

### 4.4 用户体验可测试性

4.3 节大量定性描述，缺乏客观验证手段：

| 需求 | 问题 | 建议 |
| "色彩方案使用 ui-ux-pro-max" | 外部引用不可复现 | 补充具体色板值和对比度比 |
| "波形动画/呼吸灯效果" | 无量化指标 | 补充帧率（≥30fps）和周期 |
| "错误提示使用通俗语言" | 主观 | 提供已批准的错误文案清单 + 禁止术语列表 |
| WCAG 2.1 AA | 仅提及无子项 | 至少列出色对比度 ≥4.5:1、键盘可操作、焦点可见 |

### 4.5 关键测试场景（节选）

**正常流（6 条）**：注册→登录、20 题测评定级、跟读评测逐词标记、5 轮对话评分、进度页（有数据+空状态）

**异常流（15 条）**：重复注册、密码锁定、Token 过期操作中、录音权限拒绝、来电中断、API 超时、ASR 空识别、噪音无法评分、对话中 ASR 失败、LLM JSON 解析失败、LLM 敏感内容、网络中断恢复、测评刷新丢失、录音切后台、题库不足

**边界值（31 条）**：密码 7/8/20/21 位、年龄 5/6/99、录音 0.4s/0.5s/60.0s/60.1s、音频 5.0MB/5.01MB、测评 0/40/41/70/71/100 分、单词 80/79/60/59 分、QPS 2 次/s、登录错误 5/6 次 + 锁定到期、邮箱含 `+` 号、手机号 10/11 位

> 完整测试场景表（TC-001 ~ TC-231）详见 QA 评审报告原文。

## 附录 A：补充验收标准（追加到 PRD 第 8 章）

### REQ-001 用户注册
- [ ] AC-001-1：密码 8-20 位 + 至少 1 字母 + 1 数字，前后端均校验
- [ ] AC-001-2：已注册邮箱/手机号返回 409 + "该邮箱/手机号已注册，请直接登录"
- [ ] AC-001-3：邮箱按 RFC 5322 校验（含 `+` 号合法），不超过 100 字符
- [ ] AC-001-4：同一 IP 每分钟注册请求不超过 3 次

### REQ-002 用户登录
- [ ] AC-002-1：连续 5 次密码错误锁定 30 分钟，到期自动解封
- [ ] AC-002-2：JWT 过期返回 401 + 保留页面路径回跳

### REQ-003 水平测评
- [ ] AC-003-1：每题限时 60s，超时自动跳过计 0 分，最后 10s 倒计时变红
- [ ] AC-003-2：中途刷新/关闭进度不保存，下次从头开始
- [ ] AC-003-3：题库 <20 条时展示"题库维护中，请稍后再试"

### REQ-004 发音评测
- [ ] AC-004-1：录音 <0.5s 提示"未检测到有效语音，请重新朗读"
- [ ] AC-004-2：逐词颜色：≥80 绿 / 60-79 黄 / <60 红
- [ ] AC-004-3：ASR 空文本提示"未能识别语音内容，请确保发音清晰"
- [ ] AC-004-4：无法评分时提示原因 + 建议

### REQ-005 智能对话
- [ ] AC-005-1：5 轮后自动结束，权重：语法 40% + 相关性 30% + 流利度 30%
- [ ] AC-005-2：LLM JSON 解析失败 → 正则兜底；完全失败 → 仅对话文本不评分
- [ ] AC-005-3：中间轮次失败不影响已完成轮次评分

### REQ-006 学习进度
- [ ] AC-006-1：新用户 0 条记录展示空状态占位 + 引导文案
- [ ] AC-006-2：练习次数仅统计 status=completed 的跟读和对话
- [ ] AC-006-3：无练习日期在折线图连续（值为 0，不断点）

### REQ-009 审计日志
- [ ] AC-009-1：所有 API 请求记录到 audit_logs（含耗时 ms）
- [ ] AC-009-2：audit_logs append-only，不支持 UPDATE/DELETE

### REQ-010 语音删除
- [ ] AC-010-1：文件创建满 168 小时触发物理删除（每小时轮询）
- [ ] AC-010-2：删除时写入审计日志 + 清空 audio_url 或标记 audio_deleted_at

### REQ-012 响应式
- [ ] AC-012-1：375px/768px/1024px/1440px 四断点无溢出无横向滚动
- [ ] AC-012-2：移动端触摸目标最小 44x44px（含 padding）
