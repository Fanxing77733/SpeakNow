## AI 服务接入状态

| 服务 | 状态 | 说明 |
|------|------|------|
| ASR 语音识别 | ✅ 真实（腾讯云） | 前端 WebM→WAV 自动转换，16k_en 英文识别 |
| LLM 对话 | ✅ 真实（DeepSeek） | 根据用户真实语音内容实时回复 |
| LLM 评分 | ✅ 真实（DeepSeek） | 根据真实对话内容独立评分 |
| 发音评测 | ⚠️ Mock | 暂用随机分，待接入驰声/讯飞 |
| TTS 语音播报 | ✅ 浏览器内置 | AI 回复打字完成后自动朗读，可开关 |

### 切换 AI 服务

通过 `application.yml` 中的 `aigw.asr.provider` / `aigw.llm.provider` 控制：

```yaml
# dev profile
aigw:
  asr:
    provider: mock       # 改 tencent → 真实识别
  llm:
    provider: mock       # 改 deepseek → 真实 LLM
```

真实 API Key 配置在 `application-prod.yml`（已 gitignore），不会被提交。Adapter 通过 `@ConditionalOnProperty` 自动选择加载哪个实现类。

## Bug 修复记录

| 问题 | 根因 | 修复 |
|------|------|------|
| 听力题显示文字而非播放音频 | 种子数据 transcript 内嵌在 question_text 中 | 前端对 listening 题型隐藏 transcript、Web Speech API 朗读、提供重播按钮 |
| 发音评测/对话录音后显示"服务繁忙" | `client.ts` 强制设 `Content-Type: application/json`，FormData 无法正确发送 | 去掉默认 Content-Type 头，让 Axios 自动判断 |
| WebM 录音时长校验失败 | 后端按 PCM WAV 估算时长，WebM 压缩格式被严重低估 | 前端传真实 duration 给后端 |
| Flyway 迁移冲突 | 两个 V1__ 文件 + undo 在 migration 目录 | 删重复、undo 移出 |
| JDBC 编码错误 | `characterEncoding=utf8mb4` 非 Java 编码名 | 改为 UTF-8 |
| 情景对话第 1 轮就结束 | 后端的 totalRounds:1 覆盖了 Store 默认值 5 | 不覆盖 max rounds |
| 对话页刷新后 session 丢失 | React StrictMode 双重挂载触发 resetStore | 移除 unmount 时的 store reset |
| 录音发送时 sessionId 为 null | audioBlob 状态 useEffect 时序问题 | 改为直接调用 API，不经过 state |
| 腾讯云 ASR 签名失败 | 时间戳不一致 + x-tc-action 不应在签名中 | 统一时间戳，规范签名 |
| DeepSeek 历史消息调用失败 | 数据库 role 为 "ai"，API 要求 "assistant" | 映射 ai→assistant |

## 本地运行

```bash
# 1. 启动 MySQL
net start MySQL

# 2. 启动 Redis（登录限流需要）
redis-server

# 3. 启动后端
cd h:\NLP-ES\english-speaking-backend
mvn spring-boot:run -pl es-server

# 4. 启动前端
cd h:\NLP-ES\english-speaking-frontend
npm run dev
```

打开 `http://localhost:5173`
