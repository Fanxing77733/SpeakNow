## Bug 修复记录

| 问题 | 根因 | 修复 |
|------|------|------|
| 听力题显示文字而非播放音频 | 种子数据 transcript 内嵌在 question_text 中 | 前端对 `type=listening` 题目自动隐藏 transcript、用 Web Speech API 朗读、提供重播按钮 |
| 发音评测录音后显示"服务繁忙" | `client.ts` 强制设 `Content-Type: application/json`，导致 FormData 上传时 Axios 无法自动切换 multipart | 去掉默认 Content-Type 头，让 Axios 自动判断 |
| 发音评测 WebM 时长估算错误 | 后端 AudioValidator 按 PCM WAV 估算时长，前端录的 WebM 压缩格式被严重低估 | 前端传 `useRecorder` 计时的真实 `duration` 给后端，后端优先使用 |
| 情景对话同款问题 | 同上，conversation 接口也用 AudioValidator | 同款修复 |
| Flyway 迁移冲突 | 两个 `V1__` 文件和 undo 文件版本号冲突 | 删重复文件、undo 移出 migration 目录 |
| JDBC 编码错误 | `characterEncoding=utf8mb4` 不是 Java 支持的编码名 | 改为 `characterEncoding=UTF-8` |

## 切换真实 AI API 指南

当前开发环境 (`application.yml` 中 `spring.profiles.active: dev`) 使用 Mock Adapter，无需任何 API Key。切换到真实 AI 服务需以下步骤：

### 第一步：创建真实 Adapter 实现类

在 `es-ai-gateway/src/main/java/com/es/aigw/adapter/impl/` 下创建三个实现类，标注 `@Profile("prod")`：

**1. RealAsrAdapter — 对接腾讯云 ASR**

```java
@Component
@Profile("prod")
public class RealAsrAdapter implements AsrAdapter {

    @Value("${aigw.asr.secret-id}")
    private String secretId;

    @Value("${aigw.asr.secret-key}")
    private String secretKey;

    @Override
    public String recognize(byte[] audioBytes) {
        // 1. 音频转 Base64
        // 2. 调用腾讯云 一句话识别 API
        //    https://cloud.tencent.com/document/product/1093/35646
        // 3. 返回识别文本
    }
}
```

**2. RealPronunciationEvalAdapter — 对接驰声/讯飞评测**

```java
@Component
@Profile("prod")
public class RealPronunciationEvalAdapter implements PronunciationEvalAdapter {

    @Override
    public PronunciationEvalResult evaluate(byte[] audioBytes, String referenceText) {
        // 1. 调用驰声/讯飞 发音评测 API
        // 2. 解析返回的逐词+音素结果
        // 3. 构建 PronunciationEvalResult
    }
}
```

**3. RealLlmAdapter — 对接 DeepSeek/通义千问**

```java
@Component
@Profile("prod")
public class RealLlmAdapter implements LlmAdapter {

    @Override
    public String chat(String systemPrompt, List<ChatMessage> messages, double temperature) {
        // 调用 LLM Chat Completions API
    }

    @Override
    public DialogueScoreResult scoreDialogue(String dialogueHistory) {
        // 独立评分调用，temperature=0.2
    }
}
```

### 第二步：配置 API Key

在 `application.yml` prod profile 下添加：

```yaml
aigw:
  asr:
    provider: tencent
    secret-id: ${ASR_SECRET_ID}
    secret-key: ${ASR_SECRET_KEY}
    app-id: ${ASR_APP_ID}
  pronunciation:
    provider: chivox
    app-key: ${EVAL_APP_KEY}
    secret-key: ${EVAL_SECRET_KEY}
  llm:
    provider: deepseek
    api-key: ${LLM_API_KEY}
    base-url: https://api.deepseek.com/v1
```

### 第三步：切换 Profile 启动

```bash
# 开发环境（Mock，无需 Key）
mvn spring-boot:run -pl es-server  # 默认 dev profile

# 生产环境（真实 API）
mvn spring-boot:run -pl es-server -Dspring-boot.run.profiles=prod
```

或设置环境变量后启动：

```bash
set SPRING_PROFILES_ACTIVE=prod
mvn spring-boot:run -pl es-server
```

### Adapter 模式优势

| 项目 | 说明 |
|------|------|
| 切换供应商 | 只需改配置 `aigw.xxx.provider` + 新建 Adapter 实现类 |
| 开发调试 | `@Profile("dev")` 自动注入 Mock，零依赖 |
| 编译安全 | Mock 只在 dev profile 生效，切 prod 不会误用假数据 |
| 扩展性 | 同一接口可有多个供应商实现，通过配置切换 |
